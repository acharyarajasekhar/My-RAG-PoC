## High‑Level Architecture

- **Entry Point (`Program.cs`)** – Configures the generic host, registers services, and starts `McpServerHostedService`. The host wires ASP.NET Core routing to `RpcHandler`.
- **`RpcHandler.cs`** – Implements a simple RPC endpoint (`/rpc`) that receives a user query, delegates to the RAG pipeline, and returns the generated answer.
- **RAG Pipeline**
  - **`DocumentLoader.cs`** – Reads the knowledge‑base JSON (`rag_index/kb.json`) and creates a collection of `Document` objects.
  - **`RagRetrieverFactory.cs`** – Constructs a `RagRetriever` composed of:
    - **Embedding client (`OpenAIEmbeddingClient`)** – Calls OpenRouter's `text-embedding-3-small` endpoint (see `OpenRouterService.cs`).
    - **Vector store (`FileVectorStore`)** – Persists embeddings in a lightweight file store under `./rag_index`.
    - **Retriever** – Computes cosine similarity between the query embedding and stored vectors, returning the top‑k most relevant chunks.
  - **`OpenRouterService.cs`** – Sends the retrieved context to OpenRouter's completion API and returns the final answer.
- **Configuration (`.env` + `RAG_INDEX_PATH`, `HOST_URL`, etc.)** – Environment variables are read at startup via `DotNetEnv`. Defaults are declared in `Program.cs` and can be overridden.

### Extensibility Points

| Component | What to replace/extend | Typical file to modify |
|-----------|------------------------|------------------------|
| Embedding provider | Swap `OpenAIEmbeddingClient` for another provider that implements `IEmbeddingClient`. | `RagRetrieverFactory.cs` |
| Vector store backend | Implement `IVectorStore` (e.g., SQLite, Redis) and update the factory registration. | `RagRetrieverFactory.cs` |
| RPC surface | Add additional RPC methods or authentication middleware before `RpcHandler` is added to the pipeline. | `Program.cs` / `RpcHandler.cs` |

## Project‑Specific Notes

- **Prerequisites** – .NET SDK 8.0+, OpenRouter API key.
- **Knowledge base** – Located at `RAGSharp.McpServer/rag_index/kb.json`. The format is a JSON array of document objects; updating it and restarting the server reloads the vector store.
- **Logging** – The host uses the built‑in `Microsoft.Extensions.Logging` abstractions; logs are emitted to the console by default.
- **No test suite** – The PoC does not include unit or integration tests. If you add tests, follow the standard `dotnet test` pattern and place them under a `*Tests` project.
